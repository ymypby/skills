#### 1、日志脱敏

<!-- COMMENT: Example feature section; replace or extend with actual features such as log masking, notification sending, or report generation. -->

以“通知发送组件”在记录用户手机号等敏感信息时的日志脱敏为例：

1. 在统一日志配置中声明需要脱敏的字段（如 `phoneNo`、`idCardNo` 等），并配置脱敏规则（只保留前后几位）。
2. 在统一出入参拦截器中，对打印到日志的请求参数和响应结果应用脱敏逻辑。
3. 在持久层（如 MyBatis）中，确保 SQL 日志输出时敏感字段不会以明文方式完整打印。
