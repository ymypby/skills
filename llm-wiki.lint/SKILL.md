# LLM Wiki Lint Skill

本技能指导 LLM **维护 Wiki 知识库的健康和一致性**。

## 核心理念

- **双向关系验证**：确保 references 与 referenced_by 互相匹配且描述一致
- **关系正确性验证**：通过阅读原文理解语义，验证声明的关系是否真实存在
- **循环修复机制**：阶段二用户反馈有问题或阶段三修复完成后，自动回到阶段一重新检查，持续迭代直到用户确认通过
- **知识完整性**：检测孤立页面和缺失的交叉引用
- **内容健康**：扫描矛盾和过时内容

## 三阶段循环交互流程（必须遵守）

### 阶段一：检查问题（只读模式）
- 扫描指定范围内的所有 wiki 页面
- 执行双向验证（references ↔ referenced_by）
- 执行关系正确性验证（语义验证）
- 检测孤立页面和矛盾内容
- 生成详细 Lint 报告
- **禁止在此阶段修改任何文件**

### 阶段二：用户确认（等待模式）
- 向用户展示完整的 Lint 报告
- 列出所有待修复的问题和修复计划
- **必须等待用户明确确认**
- 用户可选择：全部修复 / 部分修复 / 跳过 / 通过

### 阶段三：执行修复（写入模式）
- **仅在用户确认后执行**
- 按修复计划逐一更新文件
- 同步更新 index.md 和 log.md
- 输出最终修复结果摘要
- **修复完成后自动回到阶段一，重新检查修复结果**

## 循环机制

循环在以下情况下触发：
1. **阶段二用户反馈有问题**：用户在阶段二提出问题但不选择修复 → 自动回到阶段一重新检查
2. **阶段三修复完成**：修复完成后 → 自动回到阶段一重新检查修复结果

循环在以下条件下终止：
- 用户在阶段二明确反馈"通过"、"没问题"、"好了"等确认词

## Wiki 目录结构

```
wiki/
├── index.md               # 全局内容索引
├── log.md                 # 活动日志（追加式）
├── overview/              # 领域总体视图
├── sources/               # 大文件/重要文档摘要
├── entities/              # 领域核心实体
├── procedures/            # 流程/操作
├── concepts/              # 抽象理念/模式/原则
└── synthesis/             # 综合分析/对比报告
```

## 关键规则 - 必须遵守

1. **Lint 必须扫描指定范围内的所有页面**：用户指定单个页面、多个页面或目录时，不得跳过任何页面
2. **双向验证关系**：references 和 referenced_by 的验证同等重要，必须互相校验
3. **严格三阶段循环**：检查、确认、修复按顺序执行，禁止跳过确认环节
4. **矛盾内容必须报告**：不得自动修改，必须展示给用户判断
5. **孤立页面必须标注**：建议建立链接，但不自动修改
6. **关系正确性必须验证**：通过阅读原文理解语义，验证声明的关系是否真实存在
7. **循环直到用户确认通过**：阶段二用户反馈有问题或阶段三修复完成后，自动回到阶段一重新检查
