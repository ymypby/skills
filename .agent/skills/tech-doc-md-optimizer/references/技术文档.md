#### 1、日志脱敏

<!-- COMMENT: Example feature section; replace or extend with actual features such as log masking, notification sending, or report generation. When adding more features (2, 3, 4 ...), ensure each item describes only one independent feature or usage scenario, and provides a complete usage flow. -->

以"通知发送组件"在记录用户手机号等敏感信息时的日志脱敏为例，完整使用流程如下：

**步骤一：配置脱敏规则**

根据项目实际情况，可选择以下任一方式配置脱敏规则：

**方式一：YAML配置文件方式**

在 `application.yml` 中声明需要脱敏的字段及规则：

```yaml
logging:
  mask:
    enabled: true
    fields:
      - name: phoneNo
        pattern: "^(\\d{3})\\d{4}(\\d{4})$"
        replacement: "$1****$2"
      - name: idCardNo
        pattern: "^(\\d{4})\\d+(\\d{4})$"
        replacement: "$1********$2"
```

**方式二：Properties配置文件方式**

在 `application.properties` 中声明需要脱敏的字段及规则：

```properties
logging.mask.enabled=true
logging.mask.fields[0].name=phoneNo
logging.mask.fields[0].pattern=^(\\d{3})\\d{4}(\\d{4})$
logging.mask.fields[0].replacement=$1****$2
logging.mask.fields[1].name=idCardNo
logging.mask.fields[1].pattern=^(\\d{4})\\d+(\\d{4})$
logging.mask.fields[1].replacement=$1********$2
```

**方式三：代码配置方式**

通过 Java 配置类自定义脱敏规则：

```java
@Configuration
public class LogMaskConfig {
    @Bean
    public LogMaskProperties logMaskProperties() {
        LogMaskProperties properties = new LogMaskProperties();
        properties.setEnabled(true);
        properties.addField("phoneNo", "^(\\d{3})\\d{4}(\\d{4})$", "$1****$2");
        properties.addField("idCardNo", "^(\\d{4})\\d+(\\d{4})$", "$1********$2");
        return properties;
    }
}
```

**步骤二：引入脱敏组件**

在 Spring Boot 启动类或配置类上添加注解启用脱敏功能：

```java
@SpringBootApplication
@EnableLogMasking  // 启用日志脱敏
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

**步骤三：在业务代码中使用**

脱敏组件会自动拦截并处理日志输出，业务代码无需额外修改：

```java
@Slf4j
@Service
public class NotificationService {
    public void sendSms(String phoneNo, String content) {
        // 日志中的 phoneNo 会自动脱敏
        log.info("发送短信至手机号: {}, 内容: {}", phoneNo, content);
        // ... 发送逻辑
    }
}
```

**步骤四：验证脱敏效果**

启动应用后查看日志输出，确认敏感字段已被正确脱敏：

```
2026-02-28 10:00:00 INFO  - 发送短信至手机号: 138****1234, 内容: 您的验证码为123456
```
